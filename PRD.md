# Product Requirements Document (PRD): DJ Music Marketplace

## 1. Product Overview
**DJ Music Marketplace** is a specialized e-commerce platform designed for professional and hobbyist DJs. It serves as a hub for discovering, previewing, and purchasing high-quality music tracks, remixes, and sample packs specifically curated for DJ performances.

### Objectives
- Provide a seamless browsing experience for DJ-specific music.
- Enable high-quality audio previews with technical metadata (BPM, Key).
- Facilitate easy management of tracks for administrators.
- Support a global audience with bilingual (Thai/English) support.

---

## 2. Target Audience
- **Professional DJs:** Looking for high-quality, exclusive remixes and tracks for their sets.
- **Bedroom DJs:** Seeking affordable, curated music packs to build their libraries.
- **Music Producers/Remixers:** Who want to sell their specialized content to DJs.

---

## 3. User Stories

### For DJs (Customers)
- **As a DJ,** I want to browse tracks by genre, BPM, and key so I can find music that fits my current set.
- **As a DJ,** I want to listen to a preview of a track before buying it.
- **As a DJ,** I want to add tracks to my favorites list for later consideration.
- **As a DJ,** I want to purchase multiple tracks at once using a shopping cart.
- **As a DJ,** I want to switch between Thai and English languages for better accessibility.

### For Administrators
- **As an admin,** I want to upload new tracks with audio files and artwork.
- **As an admin,** I want to manage track metadata (BPM, Key, Genre, Version) to ensure accurate search results.
- **As an admin,** I want to delete or update existing tracks in the catalog.

---

## 4. Functional Requirements

### 4.1 Track Management
- **Catalog Browsing:** List tracks with details (Title, Artist, BPM, Key, Genre).
- **Filtering & Search:** Search by keywords and filter by genre or version type (Clean/Dirty).
- **Product Details:** Dedicated page for track/pack details including full metadata and related tracks.

### 4.2 Audio Preview System
- **Global Audio Player:** A persistent player that allows users to listen to previews while browsing.
- **Waveform Visualization:** Visual representation of the audio track.
- **Playback Controls:** Play, pause, volume control, and track switching.

### 4.3 Shopping Experience
- **Shopping Cart:** Add/remove items, view total price.
- **Favorites/Wishlist:** Save tracks to a personal list (requires authentication).
- **Checkout Process:** Placeholder for payment integration (Stripe/PayPal/PromptPay).

### 4.4 User Authentication
- **Secure Sign-in/Sign-up:** Email-based authentication via Supabase.
- **User Profile:** Manage personal information and view purchase history.

### 4.5 Admin Dashboard
- **Content Upload:** Form to upload audio files and artwork.
- **Track Management:** CRUD operations for the music catalog.
- **Statistics:** Basic overview of total tracks and user activity.

### 4.6 Internationalization (i18n)
- **Bilingual UI:** Full support for Thai and English languages across all pages.

---

## 5. Non-Functional Requirements
- **Performance:** Fast page loads and low-latency audio streaming.
- **Security:** Secure storage of user data and protected admin routes.
- **Responsiveness:** Optimized for both Desktop and Mobile devices.
- **Scalability:** Ability to handle an increasing number of tracks and users.

---

## 6. Technical Architecture
- **Frontend:** React 19, TypeScript, Vite.
- **Styling:** Custom CSS.
- **State Management:** React Context API (Auth, Audio, Cart, Favorites, Language, Notifications).
- **Backend/Database:** Supabase (PostgreSQL, Auth, Storage).
- **Icons:** Lucide React.
- **Routing:** React Router DOM.
